🎯 Algoritmos Heurísticos

Soluciones inteligentes para problemas que parecen imposibles

¿Qué son los Algoritmos Heurísticos?

Un algoritmo heurístico es un método que busca encontrar soluciones suficientemente buenas en tiempo razonable, sin garantizar que sean óptimas.

La palabra "heurística" viene del griego "heuriskein" (encontrar, descubrir). Es un arte de búsqueda inteligente.

Diferencia clave:
• Algoritmo exacto: Encuentra la solución ÓPTIMA (mejor posible), pero puede tardar mucho tiempo
• Algoritmo heurístico: Encuentra una solución muy buena en poco tiempo, pero NO garantiza que sea la mejor

¿Por qué existen?

Muchos problemas reales son NP-difíciles: su complejidad es exponencial, haciendo imposible calcular la solución óptima en tiempo práctico.

Ejemplo del Viajante de Comercio (TSP):
Con 20 ciudades: 20! ≈ 2.4 × 10¹⁸ rutas posibles
Una computadora probando 1 millón de rutas/segundo tardaría
75 millones de años en encontrar la óptima

Por eso usamos heurísticas: en 1 segundo, encuentran una ruta que es un 95% tan buena como la óptima.

¿Cuándo usar Heurísticas?

Escenario Usa Exacto (PD, Fuerza Bruta) Usa Heurístico
Tamaño del problema Pequeño (n ≤ 20) Grande (n ≥ 100)
Tiempo disponible Horas, días Segundos, minutos
Necesidad de optimalidad Crítica (necesitas la mejor) Flexible (lo suficientemente bueno)
Complejidad del problema Baja (P o NP fácil) Alta (NP-difícil)
Contexto real Científico, académico Industrial, comercial

Tipos de Algoritmos Heurísticos

1️⃣ Heurísticas Constructivas

Construyen una solución paso a paso, comenzando de cero.

Ejemplo: Nearest Neighbor (Vecino más cercano)
Para el TSP: Empieza en una ciudad, siempre ve a la ciudad no visitada más cercana, hasta visitarlas todas.
# Pseudocódigo: Nearest Neighbor para TSP def nearest_neighbor(ciudades): solucion = [] actual = ciudades[0] solucion.append(actual) no_visitadas = ciudades[1:] while no_visitadas: vecino_cercano = min(no_visitadas, key=lambda c: distancia(actual, c)) solucion.append(vecino_cercano) no_visitadas.remove(vecino_cercano) actual = vecino_cercano return solucion
✓ Ventajas: Rápido, simple, da una solución decente al instante
✗ Desventajas: Puede quedar atrapado en decisiones subóptimas

2️⃣ Heurísticas de Mejora Local (Local Search)

Comienzan con una solución cualquiera y la mejoran iterativamente.

Ejemplo: 2-Opt para TSP
Toma una ruta, intenta intercambiar pares de ciudades para ver si mejora. Si mejora, mantén el cambio. Repite hasta que no puedas mejorar más.
# Pseudocódigo: 2-Opt def two_opt(ruta): mejorado = True while mejorado: mejorado = False for i in range(len(ruta)-1): for j in range(i+2, len(ruta)): nueva_ruta = swap_edges(ruta, i, j) if costo(nueva_ruta) < costo(ruta): ruta = nueva_ruta mejorado = True return ruta
✓ Ventajas: Mejora la solución iterativamente, mejor que constructivas
✗ Desventajas: Puede quedar atrapado en óptimos locales

3️⃣ Metaheurísticas (El poder real)

Estrategias generales que guían la búsqueda para escapar de óptimos locales.

🧬 Algoritmos Genéticos (AG)

Idea: Simula evolución biológica. Tienes una población de soluciones que se reproducen y mutan.

  • Selecciona las mejores soluciones (selección natural)
  • Las combina (cruce = reproducción)
  • Introduce cambios aleatorios (mutación)
  • Repite hasta que converge

Metáfora: Las soluciones compiten como animales en la naturaleza. Los más "fuertes" se reproducen.

🌡️ Simulated Annealing (SA)

Idea: Simula el proceso físico de enfriar un metal.

  • Acepta mejores soluciones siempre
  • Acepta peores soluciones con cierta probabilidad (para escapar de óptimos locales)
  • La probabilidad disminuye gradualmente (como enfriar)

Metáfora: Cuando el metal está caliente, puede "saltar" entre estados. Al enfriarse, se queda en el estado de menor energía.

🐜 Colonia de Hormigas (ACO)

Idea: Simula cómo las hormigas encuentran rutas óptimas usando feromonas.

  • Hormigas virtuales buscan soluciones
  • Dejan "feromona digital" en buenas rutas
  • Otras hormigas siguen rastros de feromona
  • Convergen a rutas buenas

Metáfora: Las hormigas no son individuales inteligentes, pero la colonia es. Cooperación + química.

🔍 Tabu Search (TS)

Idea: Mantiene un "tabú" de soluciones visitadas recientemente para evitar ciclos.

  • Explora vecindario de solución actual
  • Se mueve a mejor vecino (incluso si empeora globalmente)
  • Marca reciente como tabú (prohibido volver)
  • Escapa de óptimos locales

Metáfora: Como un explorador que no quiere volver por donde vino.

Ejemplo Detallado: El Problema del Viajante (TSP)

El Problema

📍 5 ciudades para visitar en un viaje

Un viajero debe visitar todas las ciudades exactamente una vez y volver al origen, minimizando distancia total.

Ciudades:
  A (0, 0)
  B (1, 3)
  C (4, 2)
  D (3, 5)
  E (5, 0)

Distancias (ejemplo simplificado):

  B-C: 3.16    B-D: 2.24    B-E: 5.66
  C-D: 3.16    C-E: 1.00
  D-E: 5.00

Pregunta: ¿Cuál es la ruta más corta que visite A, B, C, D, E?

Con 5 ciudades: 5! ÷ 2 = 60 rutas a probar. Rápido.

Con 20 ciudades: 20! ÷ 2 = 1.2 × 10¹⁸ rutas. ¡IMPOSIBLE!

Solución 2: Nearest Neighbor (heurística constructiva)

1

Inicio en A

Ruta actual: [A]

2

Desde A, ¿quién es el más cercano?

A-B: 3.16 ✓ CERCANO
A-C: 4.47
A-D: 5.83
A-E: 5.00
Elegir B. Ruta: [A, B]

3

Desde B, ¿quién es el más cercano (no visitado)?

B-C: 3.16
B-D: 2.24 ✓ MÁS CERCANO
B-E: 5.66
Elegir D. Ruta: [A, B, D]

4

Desde D, ¿quién es el más cercano (no visitado)?

D-C: 3.16 ✓
D-E: 5.00
Elegir C. Ruta: [A, B, D, C]

5

Solo queda E, elegir E

Ruta: [A, B, D, C, E]

6

Cerrar el ciclo, volver a A

Ruta final: A → B → D → C → E → A

Distancia total: 3.16 + 2.24 + 3.16 + 1.00 + 5.00 = 14.56

✓ Resultado: Solución decente en segundos (no garantiza ser óptima, pero es rápido)

3

Al final converge a una solución muy buena (probablemente óptima en este caso pequeño)

Aplicaciones Reales en Ingeniería

Optimizar rutas de reparto para miles de paquetes. Usan Genetic Algorithms y Simulated Annealing para minimizar gasolina y tiempo.

Impacto: Ahorran millones en combustible anualmente.

🏭 Planificación de Producción

Asignar tareas a máquinas en una fábrica minimizando makespan (tiempo total). Problema NP-difícil.

Impacto: Aumentan productividad 15-20%.

Asignar vuelos a puertas, minimizar cambios de avión. Problema de asignación cuadrática.

Impacto: Reducen retrasos, mejoran experiencia.

🧠 Redes Neuronales

Usar AG para optimizar pesos de red neuronal (alternativa a backpropagation para ciertos problemas).

Impacto: Entrenan modelos más robustos en algunos escenarios.

🌐 Diseño de Redes Telemáticas

¿Dónde colocar servidores para minimizar latencia? Problema NP-difícil en grafos grandes.

Impacto: Mejoran performance de internet.

🔌 Distribución de Energía

Ruteo óptimo de líneas eléctricas minimizando pérdidas. ACO es especialmente bueno aquí.

Impacto: Reducen desperdicio de energía.

Ventajas vs Desventajas

Aspecto ✓ Ventajas ✗ Desventajas
Velocidad Encuentran soluciones en segundos No garantizan optimalidad
Escalabilidad Manejan problemas grandes (n > 1000) Calidad degrada con tamaño
Flexibilidad Se adaptan a restricciones complejas Tuning de parámetros es difícil
Implementación Relativamente fáciles de codificar Comportamiento impredecible (aleatorios)
Garantías teóricas Muchos tienen base matemática sólida No garantizan convergencia (algunos)

La pregunta clave: ¿Cuándo son "suficientemente buenas"?

En la mayoría de problemas industriales, una solución que sea 95% óptima en 1 segundo es infinitamente mejor que una que sea 100% óptima en 1000 años.

Eso es el poder de las heurísticas: pragmatismo matemático.
