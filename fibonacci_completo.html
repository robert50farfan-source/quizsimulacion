🔄 Fibonacci: Recursividad vs Programación Dinámica

Entiende por qué la PD es millones de veces más rápida

Recursividad Ingenua ⚠️

El enfoque más obvio, pero extremadamente ineficiente. Recalcula los mismos valores una y otra vez.

def fib_recursivo(n): # Caso base if n <= 1: return n # Caso recursivo return fib_recursivo(n-1) + \ fib_recursivo(n-2) # Ejemplo resultado = fib_recursivo(5) # Resultado: 5
⚠️ Problema: Para calcular fib(5), calcula fib(3) dos veces, fib(2) tres veces, fib(1) cinco veces... Millones de cálculos repetidos.

Programación Dinámica ✨

El enfoque inteligente. Guarda resultados para evitar recalcular.

def fib_dinamico(n): # Crear tabla para guardar resultados dp = [0] * (n + 1) # Casos base dp[0] = 0 if n >= 1: dp[1] = 1 # Llenar tabla de abajo hacia arriba for i in range(2, n + 1): dp[i] = dp[i-1] + dp[i-2] return dp[n] # Ejemplo resultado = fib_dinamico(5) # Resultado: 5
✓ Ventaja: Cada valor se calcula UNA SOLA VEZ. Usa una tabla para guardar resultados previos.

🟢 Programación Dinámica: fib(5)

Aquí llenamos una tabla de izquierda a derecha, cada valor una sola vez:

1

Crear la tabla

Declaramos una lista dp = [0, 0, 0, 0, 0, 0] (índices 0 a 5)

2

Casos base

dp[0] = 0, dp[1] = 1
Tabla ahora: [0, 1, 0, 0, 0, 0]

3

Llenar i=2

dp[2] = dp[1] + dp[0] = 1 + 0 = 1
Tabla: [0, 1, 1, 0, 0, 0]

4

Llenar i=3

dp[3] = dp[2] + dp[1] = 1 + 1 = 2
Tabla: [0, 1, 1, 2, 0, 0]

5

Llenar i=4

dp[4] = dp[3] + dp[2] = 2 + 1 = 3
Tabla: [0, 1, 1, 2, 3, 0]

6

Llenar i=5 (¡final!)

dp[5] = dp[4] + dp[3] = 3 + 2 = 5
Tabla final: [0, 1, 1, 2, 3, 5]
✓ Respuesta: dp[5] = 5

✓ Total de operaciones: 5 iteraciones
Para fib(10): 10 iteraciones
Para fib(20): 20 iteraciones
Para fib(35): 35 iteraciones (vs 29.8 millones en recursividad)

Bonus: Optimizar Espacio en PD

La solución de PD usa O(n) espacio para la tabla. ¿Podemos hacerlo mejor? Sí, observa que fib(i) solo depende de fib(i-1) e fib(i-2):

def fib_dinamico_optimizado(n): # Solo guardar los últimos DOS valores if n <= 1: return n prev2 = 0 # fib(i-2) prev1 = 1 # fib(i-1) for i in range(2, n + 1): actual = prev1 + prev2 prev2 = prev1 prev1 = actual return prev1 # Ejemplo: fib_dinamico_optimizado(5) = 5
✓ Ventaja: Complejidad espacial reducida de O(n) a O(1). Mismo tiempo O(n), pero usando solo dos variables en lugar de una tabla.

Resumen: La Lección

A

Recursividad ingenua es intuitiva pero lenta

Genera un árbol de llamadas que recalcula los mismos valores exponencialmente. Funciona para n pequeños (≤30).

B

Programación Dinámica evita recalculos

Guarda resultados en una tabla y los reutiliza. Transforma O(2ⁿ) en O(n).

C

Puedes optimizar aún más

Si solo necesitas los últimos dos valores, usa O(1) espacio en lugar de O(n).

D

Aplica esto a cualquier problema recursivo

Mochila, caminos, alineamiento de ADN... Si hay subproblemas solapados, la PD acelera exponencialmente.
