El problema central

La explosión combinatoria

Muchos problemas de informática tienen un número de soluciones que crece exponencialmente. Calcular Fibonacci(50) de forma ingenua hace más de mil millones de llamadas repetidas.

  • 1Subestructura óptima: la solución óptima se construye con soluciones óptimas de subproblemas.
  • 2Subproblemas solapados: los mismos subproblemas aparecen una y otra vez.
  • 3La idea PD: resuelve cada subproblema una sola vez y guarda el resultado (memoización / tabulación).

El corazón teórico

Principio de optimalidad de Bellman

“Una política óptima tiene la propiedad de que, sean cuales sean el estado y la decisión iniciales, las decisiones restantes deben constituir una política óptima respecto al estado resultante.”

En lenguaje humano

Si la ruta más corta de A a C pasa por B, entonces el tramo de B a C es también la ruta más corta de B a C. No puede haber un atajo mejor "escondido" dentro de un óptimo.

Esto permite escribir una ecuación recursiva (ecuación de Bellman) que define el valor óptimo en función de valores óptimos más pequeños.

Ecuación genérica 
V(estado) =
   opt [ costo(decisión)
        + V(estado_siguiente) ]
   sobre todas las
   decisiones válidas

Donde opt es min (costos) o max (ganancias).

De la teoría al código

Mochila 0/1 en Python

# Programación dinámica — tabulación
def mochila(pesos, valores, W):
    n = len(pesos)
    dp = [[0]*(W+1) for _ in range(n+1)]

    for i in range(1, n+1):
        for w in range(W+1):
            # opción A: no tomar el objeto i
            dp[i][w] = dp[i-1][w]
            # opción B: tomarlo (si cabe)
            if pesos[i-1] <= w:
                dp[i][w] = max(
                    dp[i][w],
                    valores[i-1]
                    + dp[i-1][w-pesos[i-1]]
                )
    return dp[n][W]

Lectura de la recurrencia

  • APara cada objeto comparamos no incluirlo (heredar la fila de arriba)…
  • B…contra incluirlo: su valor + el óptimo del peso restante.
  • ✓Tomamos el máximo. La celda final dp[n][W] es la respuesta óptima global.
Complejidad

Tiempo O(n·W) — pseudopolinomial. La fuerza bruta sería O(2ⁿ). Con 30 objetos eso son +1.000 millones de combinaciones vs. una tabla manejable.

El giro de la unidad

Cuando ni la PD alcanza

Problemas como el enrutamiento de vehículos (VRP) o el viajante de comercio (TSP) son NP-difíciles: no se conoce ningún algoritmo exacto eficiente para tamaños grandes.

  • !El TSP con 20 ciudades tiene ~10¹⁷ rutas posibles.
  • ≈La respuesta práctica: algoritmos heurísticos que buscan soluciones muy buenas en tiempo razonable, sin garantizar la óptima.

AG

Algoritmos Genéticos — "evolucionan" poblaciones de soluciones mediante cruce y mutación.

SA

Simulated Annealing — acepta empeoramientos al inicio para escapar de óptimos locales.

ACO

Ant Colony — hormigas virtuales depositan "feromona" en buenas rutas.
